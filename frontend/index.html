Hướng dẫn sử dụng

Train YOLO bằng GUI từ đầu đến cuối

Trang này giải thích các mục trong app bằng ngôn ngữ dễ hiểu. Nếu gặp từ chuyên môn, rê chuột vào biểu tượng dấu hỏi bên cạnh nhãn để xem giải thích nhanh.

1

Chuẩn bị dữ liệu

Chọn thư mục dataset, nhập danh sách nhãn và để GUI tạo file cấu hình cần thiết cho YOLO.

2

Kiểm tra môi trường

Nếu máy thiếu PyTorch, CUDA hoặc Ultralytics, dùng các nút cài trong GUI. Log cài đặt hiện ngay trong app.

Chọn dataset, kiểu model và mức huấn luyện. Người mới nên giữ mức Cân bằng và Máy chạy Tự động.

Lời khuyên

Nên chọn gì khi mới bắt đầu

Model: chọn bản nhanh hoặc nhỏ trước để kiểm tra dataset. Khi mọi thứ ổn mới tăng lên model lớn hơn.

Mức huấn luyện: chọn Cân bằng. Test nhanh chỉ dùng để phát hiện lỗi dữ liệu, Train kỹ dùng khi dataset đã ổn.

Máy chạy: để Tự động. Chỉ chọn GPU khi chắc CUDA đã sẵn sàng, chọn CPU khi muốn tương thích dù chậm hơn.

Automation: dùng Full pipeline khi muốn GUI tự chạy toàn bộ luồng, nhưng nên thử Chuẩn bị dataset trước nếu đây là dataset mới.

Cloud Storage: bật khi muốn lưu full log/config/output theo project. Đặt tên project dễ nhớ trước khi train để sau này mở lại đúng dữ liệu.

Cloud workspace

Google Drive dùng chung cho mọi máy

Bật Cloud để GUI dùng một chuẩn thư mục chung. Khi local hoặc Colab kết nối cùng Google Drive folder, dữ liệu sẽ được map về cùng cấu trúc.

Chưa bật
  1. 1Bật CloudCho phép dùng workspace chung
  2. 2Kiểm tra keyXác nhận Cloud API key
  3. 3Drive AuthCấp quyền Google Drive
  4. 4Tạo workspaceFolder chuẩn và project

Bật Cloud mode rồi kiểm tra Cloud key để mở bước Google Drive.

Bước 1 Cloud key

Key này chỉ dùng để kiểm tra Google Cloud/Drive API đã sẵn sàng. Nó không phải quyền truy cập Drive.

Cloud API key chỉ lưu local trong logs/cloud/ hoặc lấy từ env YOLO_GUI_GOOGLE_API_KEY; không được commit lên GitHub.

Bước 2 Google Drive Auth

Dán OAuth access token có quyền Drive. GUI dùng token này để tự tạo workspace, folder chuẩn và project.

Bước 3 Quy chuẩn lưu trữ
Tạo workspace Drive GUI sẽ tự tạo datasets, models, runs, annotations, configs, exports, logs và projects.
