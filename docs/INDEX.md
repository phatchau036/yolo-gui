# Docs Index

Đọc theo thứ tự này khi dev mới tiếp quản:

1. `README.md`: mục tiêu sản phẩm, cách chạy nhanh.
2. `docs/REPO_MAP.md`: file/folder nào làm gì.
3. `docs/PROJECT_LOGIC.md`: luồng train YOLO từ UI tới subprocess.
4. `docs/CODE_STYLE.md`: quy tắc đặt tên, code style, cách thêm logic.
5. `docs/LOGGING_AND_DEBUGGING.md`: log nằm ở đâu, debug theo job thế nào.
6. `docs/BUG_FIX_PLAYBOOK.md`: lỗi thường gặp và cách xử lý.
7. `docs/TODO_AND_STATUS.md`: đã xong gì, còn cần làm gì.
8. `docs/PROJECT_MEMORY.md`: quyết định dài hạn để agent/dev sau không phải hỏi lại.
9. `docs/SESSION_LOG.md`: nhật ký phiên làm việc mới nhất.

Quy tắc duy trì: mọi thay đổi làm đổi hành vi train, API, UI hoặc logging đều phải cập nhật ít nhất một file docs tương ứng.
